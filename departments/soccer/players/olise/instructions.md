# サッカー選手ウォッチ部門 - Michael Olise(オリーセ)

## 目的
毎朝、フランス代表 Michael Olise(マイケル・オリーセ)選手について、直近の試合予定と新着トピックを調査してレポートを作成する。

## 前提(重要)
所属クラブ・代表歴などは固定情報として扱わない。移籍市場のタイミング次第で所属が変わっている可能性があるため、**毎回の調査で最新の所属クラブ・所属状況をWeb検索で確認してから**調査を進める。

## 対象範囲
- 次の試合予定(所属クラブ・代表いずれも対象)
- 試合出場・成績(出場有無、ゴール、アシストなど)
- 移籍・契約に関する情報(移籍報道、契約延長、噂話を含む)
- 私生活関連の話題(インタビュー、SNSでの発言、スポンサー契約など)

## 調査方法
- Web検索(英語メディア中心)で選手の現在の所属を確認したうえで、直近24〜48時間以内の情報を収集する
- 次の試合予定は所属クラブの公式サイト・信頼できるスポーツメディアで確認する
- 移籍報道は噂の段階でも信頼度の高いメディア(The Athletic, Fabrizio Romano, BBC Sport, Sky Sports等)発であれば採用し、情報源の信頼度が低い場合はその旨を明記する
- 同一トピックの重複報道は1つに統合する
- 情報源のURLを必ず記録する

## 出力先
`departments/soccer/players/olise/reports/YYYY-MM/YYYY-MM-DD.md`(月ごとにフォルダを分け、その中に1日1ファイル)

## 出力フォーマット
- 「概要・背景・影響」のような分割はせず、トピックごとに1〜3文の要約+情報源のみとする(シンプル優先)
- 各トピックの見出しには通し番号を付ける
- 「次の試合予定」は毎日必ず書く(直近1〜2試合分)。新着トピックは無ければ「新着トピックなし」の1行のみでよい

```md
# Olise調査レポート - YYYY-MM-DD

## 次の試合予定
[日付] [対戦相手] ([大会名])
例: 2026-07-10 vs レアル・マドリード(CLベスト4)

## 新着トピック

### 1. [トピック1の見出し]
1〜3文程度の要約。
[情報源](URL)

### 2. [トピック2の見出し]
...
```

新着が無い場合:
```md
# Olise調査レポート - YYYY-MM-DD

## 次の試合予定
[日付] [対戦相手] ([大会名])

## 新着トピック
新着トピックなし。
```

## 優先度・件数の目安
- 件数の上限は設けない(新情報がある分だけ書く)。無理にトピックを作らない
